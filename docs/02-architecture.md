# 02) المعمارية العامة

## مبدأ التصميم
- Super App Shell بواجهة موحدة وهوية دخول واحدة.
- Microservices مستقلة عبر API Gateway مركزي.
- ملف الطالب الموحد هو مصدر الحقيقة (Source of Truth).

## طبقات النظام
- **الواجهة**: تطبيق موحد متعدد الأدوار.
- **البوابة**: API Gateway للمصادقة، التوجيه، الحماية، والـ rate limit.
- **الخدمات**: Identity, Yara, Live Library, Student Profile, Parent Dashboard.
- **البيانات**: مخازن مستقلة لكل خدمة + مخزن ملف الطالب الموحد.
- **الحدثية**: Event Bus لتبادل الأحداث بين الخدمات.

## تدفق عالي المستوى
1. المستخدم يسجل الدخول عبر Identity.
2. الطلبات تمر عبر API Gateway.
3. كل خدمة تنفذ منطقها وتصدر أحداثًا موحدة.
4. Student Profile يستهلك الأحداث ويحدّث ملف الطالب.
5. Parent Dashboard يقرأ المؤشرات المجمعة من Student Profile.
